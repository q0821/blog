SEO 策略

  • 大量內容授權與聚合（多元性、形式統一）
  • 拆解內容關鍵字（為了分類，整理讓瀏覽者方便瀏覽）
  • 2個資料庫做演算法串連管理（內部連結、關鍵字、各種連結）
  • 外部流量與關聯連結（大量使用者、瀏覽時間、瀏覽頻率，他們有跟中華電信free wifi合作，要用免費wifi 就會先開字媒體）
  • 多角度的內容組合面向（懶人包、一對多、多對一）

但是這作法…一般網站經營者根本無法自動化做這些動作，還有流量的導入根本就…，要人工做非常辛苦啊～～～～

子佈景

  • 外掛方式
    • Child Theme Configurator（有中文，照步驟做就可以了）
    • https://tw.wordpress.org/plugins/child-theme-configurator/
  • 手動方式
    • 新增 /wp-content/themes 子佈景的資料夾
    • 建立 function.php
    • 建立 style.css
    • style.css 註解
    • 參考資料 https://codex.wordpress.org/Child_Themes

自訂頁面版面

  • 在子佈景資料夾裡面建立 page-[名稱].php
  • 上面的檔案裡面最上面加上 <?php / Template Name: Template 名稱 / ?>
  • 參考資料：https://www.wpbeginner.com/wp-themes/how-to-create-a-custom-page-in-wordpress/

維護管理

  • 後台項目管理
    • 利用外掛隱藏後台項目： Admin Menu Editor （https://tw.wordpress.org/plugins/admin-menu-editor/）
  • WordPress Hook
    • Action Hook（通常不需要有回傳）
    • Filter Hook（通常需要有回傳）
    • 講者投影片裏黃色是必填的，橘色是選填
  • 忘記管理員密碼
    • 直接進資料庫 選MD5，直接輸入密碼
    • 硬上他！
  • 管理權限
    • add_filiter / user_has_cap，過濾使用者權限
  • 禁止編輯檔案修改
    • wpconfig.php DISALLOW_FILE_EDIT true
    • https://codex.wordpress.org/Editing_wp-config.php
  • hook filter 去哪找
    • Wordpress codex

1秒鐘的奮戰：WordPress 網站優化的愛恨情仇

WilliamTai
https://www.williamtai.moe/

  • 測試工具 Pingdom Website Speed Test
  • 慢是怎麼回事
    • 伺服器端改善
      • HTTP/2
      • https://http2.akamai.com/demo
    • WordPress 網站內部
      • 快取外掛
        • W3 Total Cache：設定複雜、要反覆測試，才能確保都功能正常。
        • WP Super Cache：安全，該有都有
        • Cache Enabler：設定陽春、支援WebP格式
      • WebP圖檔格式
        • Google 釋出的圖片格式
        • Chrome、Opera 支援
        • EWWW Image Optimizer 外掛支援 JPG to WebP
    • CDN
      • CloudFlare
        • https://www.cloudflare.com/products/argo-smart-routing/
        • 小課金： Argo（月費制：USD$5 + 使用量）
        • Time to first Byte （TTFB）改善約30%；
  • WordPress 網站最佳化甘苦談
    • WebP 格式不是所有瀏覽器都支援
    • 外部的JS檔案
    • WordPress 核心/外掛/佈景升級
    • 測試結果不等於使用者實際體驗
  • 最佳化原則
    • 核心檔案不要改
    • 功能加減以外掛/佈景為單位
    • 分散伺服器負擔
  • 結論
    • 到底是前端畫面慢、還是根本就是後段 query 就慢了。
    • 保持系統最佳化跟擴充性的平衡
    • 沒有唯一解
